병렬 개발 원칙

A는 실제 Feature, B는 Mock 기반 판정, C는 Mock JSONL 기반 평가를 동시에 개발한다.

Threshold 반영

C 분석·추천 → B Rule 반영 → 공동 재검증

오늘 완료 기준

실시간 Feature → V0/V2 판정 → JSONL 저장 → Replay → 자세별 일치율과 추천 threshold 출력
공통 타입 변경은 세 명이 합의한 경우에만 진행하고, 각 담당자는 소유 경로 밖의 파일을 직접 수정하지 않는다.
